# 🍜 餐厅管理系统开发指南（0 → 1 单人开发版）

---

# 1. 文档目的

本指南用于帮助你从 0 到 1 构建一个餐厅管理系统，明确：

- 开发整体流程
- 每个阶段需要做什么
- 写代码前必须准备的内容
- 如何避免常见坑

---

# 2. 完整开发流程
想法 → 范围定义 → 业务流程 → PRD → 页面结构 → 数据模型 → 数据库设计 → 技术方案 → 开发 → 测试 → 上线 → 迭代


---

# 3. 写代码前必须完成的内容（非常重要）

在开始开发之前，你至少需要完成以下 8 项：

- [ ] 产品范围（Scope）
- [ ] MVP 定义（第一版做什么/不做什么）
- [ ] 核心业务流程图
- [ ] 用户角色与权限
- [ ] 功能模块清单
- [ ] PRD 文档
- [ ] 页面结构（Sitemap）
- [ ] 数据模型设计

⚠️ 如果没有这些直接写代码，后期一定会推翻重做。

---

# 4. 第一阶段：产品设计（Product Design）

---

## 4.1 定义项目范围（Scope）

### 你要回答的问题：

- 系统给谁用？
- 解决什么问题？
- 第一版做哪些功能？
- 第一版不做哪些功能？

---

### 示例（你的项目）

#### 目标用户
- 北美中式快餐 / 面馆 / 简餐店

#### 第一版要做
- 点单系统（POS）
- 后厨接单（KDS）
- 出餐管理
- 基础库存
- 基础报表

#### 第一版不做
- AI 广告
- 工资系统
- 会员系统
- 多仓库复杂逻辑

---

## 4.2 梳理核心业务流程（最重要）

你至少需要画 4 个流程：

---

### 1️⃣ 点单流程
顾客下单 → 前台录单 → 提交订单 → 发送到厨房

---

### 2️⃣ 后厨流程
订单进入 → 自动拆分工位 → 各工位制作 → 组装 → Ready

---

### 3️⃣ 出餐流程
订单 Ready → 显示取餐 → 顾客取餐 → 完成

---

### 4️⃣ 库存流程
订单完成 → BOM 扣库存 → 检查库存 → 低库存提醒

---

📌 为什么必须先做流程：
- 所有页面、数据库、接口都来自流程
- 流程错了，系统就错了

# 5. 第二阶段：PRD（产品需求文档）

---

## PRD 应该包含：

每个功能都需要写清楚：

- 功能目的
- 使用角色
- 操作流程
- 系统行为
- 业务规则
- 异常情况

---

### 示例：创建订单

#### 功能说明
前台创建订单

#### 规则
- 必须至少一个菜品
- 可添加备注
- 支持堂食/自取
- 提交后自动进入厨房

#### 状态变化

Draft → Submitted → In Preparation
